# Threat Modeling: Designing for Security
**Author:** Adam Shostack

## What This Book Is About
Threat Modeling: Designing for Security is the definitive guide to threat modeling for software and systems. Shostack presents a four-step framework centered on four questions: What are you building? What can go wrong? What should you do about it? Did you do a decent job? The book covers attacker-centric, asset-centric, and software-centric approaches, with practical guidance for integrating threat modeling into the development lifecycle.

## Key Topics
- The four-step threat modeling framework
- STRIDE and other threat categorization methods
- Mapping attack surfaces and data flows
- Attacker-centric vs. software-centric approaches
- Integrating threat modeling into software development

## When To Reach For This Book
When you need to systematically identify and address security threats in software or system design.
